## Criar transação

```http
POST /api/v1/transactions
```

### Request body

| Campo            | Tipo     | Obrigatório | Descrição                                      |
|------------------|----------|-------------|-------------------------------------------------|
| `amount`         | decimal  | Sim         | Valor em reais (ex: `150.00`)                  |
| `paymentType`    | string   | Sim         | `PIX`, `CREDIT_CARD` ou `DEBIT`               |
| `installments`   | int      | Não         | Parcelas (apenas cartão, default: 1)           |
| `sellerId`       | uuid     | Não         | ID do seller (sub-conta) para split            |
| `description`    | string   | Não         | Descrição da cobrança                          |
| `idempotencyKey` | string   | Sim         | Chave única para evitar duplicidade            |
| `externalReferenceId` | string | Não    | Referência externa para deduplicação           |
| `splits`         | array    | Não         | Lista de splits manuais (ver abaixo)           |
| `splitRuleId`    | uuid     | Não         | ID de uma regra de split reutilizável          |
| `feeAllocationPolicy` | string | Não    | Quem paga as taxas: `PRIMARY_SELLER_PAYS_FEES` (default), `PROPORTIONAL_TO_RECIPIENTS`, `PLATFORM_ABSORBS` |

### Exemplo

```bash
curl -X POST /api/v1/transactions \
  -H "Content-Type: application/json" \
  -H "X-Api-Key: {apiKey}" \
  -d '{
    "amount": 250.00,
    "paymentType": "PIX",
    "sellerId": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
    "description": "Pedido #42",
    "idempotencyKey": "order-42"
  }'
```

### Response `201 Created`

```json
{
  "id": "uuid",
  "status": "PENDING",
  "amount": 250.00,
  "gateway": {
    "transactionId": "fellow-tx-id",
    "qrCodeBase64": "data:image/png;base64,...",
    "qrCodeText": "00020126...",
    "paymentUrl": "https://pay.fellowpay.com.br/..."
  }
}
```

### Objeto `splits[]`

| Campo        | Tipo    | Obrigatório | Descrição                          |
|-------------|---------|-------------|------------------------------------|
| `sellerId`  | uuid    | Sim         | ID do seller recebedor             |
| `amount`    | decimal | Não*        | Valor fixo em reais                |
| `percentage`| decimal | Não*        | Porcentagem do valor líquido       |

\* Informe `amount` ou `percentage`, não ambos. Não é possível enviar `splits` e `splitRuleId` simultaneamente.

### Limites de split por plano

| Plano   | Máximo de recebedores |
|---------|----------------------:|
| COMECE  | 2                     |
| CRESCA  | 10                    |
| SCALA   | 50                    |

### Margem na transação (detalhe)

O endpoint `GET /api/v1/transactions/{id}` retorna os campos de margem:

| Campo                  | Descrição                                    |
|------------------------|----------------------------------------------|
| `platformFeeAmount`    | Taxa cobrada do seller (FellowCore)          |
| `providerCostAmount`   | Custo real do provider (Stripe/OpenPix)     |
| `platformMarginAmount` | Margem: platformFee - providerCost          |

## Reembolsar transação

```http
POST /api/v1/transactions/{id}/refund
```

### Request body

| Campo    | Tipo    | Obrigatório | Descrição                                       |
|----------|---------|-------------|-------------------------------------------------|
| `amount` | decimal | Não         | Valor do reembolso. Se omitido, reembolso total. |
| `reason` | string  | Não         | Motivo do reembolso.                            |

:::tip[Reembolso parcial]
Você pode fazer múltiplos reembolsos parciais até o valor total da transação. A cada chamada, `refundedAmount` é acumulado. Quando `refundedAmount >= amount`, o status muda para `REFUNDED`.
:::

### Exemplo — reembolso parcial

```bash
curl -X POST /api/v1/transactions/{id}/refund \
  -H "Content-Type: application/json" \
  -H "X-Api-Key: {apiKey}" \
  -d '{
    "amount": 50.00,
    "reason": "Produto avariado"
  }'
```

### Response `200 OK`

```json
{
  "id": "uuid",
  "status": "CAPTURED",
  "refundId": "refund-correlation-id"
}
```

### Regras de negócio

- Apenas transações com status `CAPTURED` podem ser reembolsadas.
- O valor do reembolso deve ser maior que zero.
- A soma dos reembolsos não pode exceder o valor original da transação.
- Em caso de concorrência, a API retorna `409 Conflict` — basta tentar novamente.

### Status da transação

| Status       | Descrição                                      |
|--------------|-------------------------------------------------|
| `PENDING`    | Criada, aguardando pagamento                   |
| `CAPTURED`   | Pagamento confirmado                           |
| `EXPIRED`    | Expirou sem pagamento                          |
| `REFUNDED`   | Totalmente reembolsada                         |
| `FAILED`     | Falha no processamento                         |

## Cancelar transação

```http
POST /api/v1/transactions/{id}/cancel
```

Cancela uma transação pendente. Se a transação for do tipo Pix, a cobrança é cancelada junto ao provedor.

### Response `204 No Content`

### Regras de negócio

- Apenas transações com status `CREATED` podem ser canceladas.

---

## Atualizar expiração

Atualiza a data de expiração de uma cobrança Pix pendente.

```http
PATCH /api/v1/transactions/{id}
```

### Request body

| Campo       | Tipo     | Obrigatório | Descrição                        |
|-------------|----------|-------------|----------------------------------|
| `expiresAt` | datetime | Sim         | Nova data de expiração (ISO 8601) |

### Exemplo

```bash
curl -X PATCH /api/v1/transactions/{id} \
  -H "Content-Type: application/json" \
  -H "X-Api-Key: {apiKey}" \
  -d '{ "expiresAt": "2026-05-01T23:59:59Z" }'
```

### Response `200 OK`

Retorna o mesmo formato de `TransactionDetailDto`.

### Regras de negócio

- Apenas transações Pix com status `CREATED` podem ter a expiração atualizada.

## Consultar reembolso

```http
GET /api/v1/transactions/{id}/refunds/{refundId}
```

### Response `200 OK`

Retorna um único objeto `RefundDetailDto` com o mesmo formato da listagem.

## Comprovante de transação

```http
GET /api/v1/transactions/{id}/receipt?type=payment
```

### Query parameters

| Parâmetro | Tipo   | Obrigatório | Descrição                    |
|-----------|--------|-------------|------------------------------|
| `type`    | string | Sim         | Tipo do comprovante (ex: `payment`) |

### Response `200 OK`

Retorna o arquivo PDF do comprovante (`application/pdf`).

### Regras de negócio

- Apenas transações Pix com status `CAPTURED` possuem comprovante.
